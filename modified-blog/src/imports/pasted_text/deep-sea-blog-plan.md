심해·고래 테마 블로그 개발 계획서
1. 프로젝트 개요

새로운 블로그는 심해 속으로 ‘잠수하는’ 느낌을 전달하면서 고래와 해양 생명체를 주제로 콘텐츠를 소개하는 것을 목표로 합니다. 드래그나 스크롤에 따라 사용자에게 바닷속을 탐험하는 듯한 경험을 제공하고, 각 섹션에 자신의 경력과 프로젝트를 카드 형태로 보여줍니다. 본 계획서는 전체 개발 흐름과 세부 구현 방법을 제시합니다.

2. 핵심 기능 및 디자인 포인트
입장 및 로딩 화면 – 방문자가 페이지에 들어오면, Killian Herzer 포트폴리오처럼 초기화 화면과 함께 로딩 퍼센트를 표시하는 장면을 구현합니다. 고래 모양 아이콘과 함께 “Dive In” 메시지를 보이고, 로딩 완료 후 본문으로 전환합니다.
심해로 내려가는 스크롤 – Neal Agarwal의 Deep Sea 사이트처럼 스크롤에 따라 해양 생물을 보여주며 심해로 내려가는 연출을 반영합니다. 사용자의 경력·프로젝트를 각각 다른 깊이에 배치해 스토리텔링을 강화합니다.
고래 애니메이션 – CodeTap의 튜토리얼처럼 HTML/CSS/JS 기반의 고래 애니메이션을 제작해 커서나 스크롤에 반응하도록 합니다. 생동감 있는 고래가 블로그 곳곳에서 등장해 분위기를 돋웁니다.
카드형 정보 디스플레이 – 각 프로젝트나 경력 항목은 카드 형식으로 나타나며, 스크롤할 때 카드가 서서히 나타나고 확대되는 애니메이션을 적용합니다. 이때 카드 배경과 글씨 색은 해저 분위기에 맞춰 조정합니다.
진행 막대 및 네비게이션 – 사용자가 어느 깊이에 있는지 보여주는 세로 진행 막대를 우측에 표시합니다. One Page Love의 사례처럼 스크롤 진행률을 보여주어 긴 페이지에서의 위치를 인지할 수 있게 합니다.

4. 개발 단계
4.1 디자인과 자료 수집
레퍼런스 조사 – Awwwards 수상작 및 해양 테마 웹사이트를 추가로 조사해 색감·폰트·구성 아이디어를 정리합니다. 디자인모도 기사에서 언급되듯 자연적인 해양 풍경이나 사진을 활용해 사용자의 스크롤을 유도하는 것이 효과적입니다.
와이어프레임 작성 – 페이지의 주요 섹션(로딩 화면, 소개, 경력, 프로젝트, 컨택) 배치와 스크롤 흐름을 Figma로 설계합니다.
컬러 팔레트 및 타이포그래피 – 해양 블루와 검푸른 색을 주색으로, 고대비를 위해 밝은 글씨와 포인트 색을 정합니다. 폰트는 현대적이고 읽기 쉬운 sans-serif 계열을 선정합니다.
4.2 프론트엔드 개발
프로젝트 셋업 – Vite를 이용해 React 프로젝트를 생성하고 Tailwind CSS를 설정합니다.
공통 컴포넌트 개발
<Loader>: 로딩 퍼센트 애니메이션과 고래 아이콘을 표시하는 컴포넌트. 이미지/글꼴 사전 로딩 로직을 구현합니다.
<ParallaxSection>: 각 심해 섹션을 래핑하며 ScrollTrigger로 배경 위치를 이동하는 컴포넌트. 배경 이미지는 depth에 따라 색상을 점진적으로 바꿉니다.
<WhaleAnimation>: SVG 혹은 Lottie 애니메이션을 렌더링하며, 커서 위치나 스크롤 값을 기반으로 움직입니다.
<Card>: 프로젝트·경력 정보를 표시하는 재사용 가능한 컴포넌트. GSAP를 이용해 등장 애니메이션을 부여합니다.
<ProgressBar>: 스크롤 위치를 감지하여 세로 막대의 높이를 조정합니다.
스크롤 및 인터랙션 로직
ScrollTrigger를 사용해 각 섹션이 viewport에 들어올 때 애니메이션을 실행합니다.
Parallax 효과를 위한 배경 이동, 블러 처리, 투명도 변화를 구현합니다.
콘텐츠 통합
자신의 이력/경력 데이터와 프로젝트 정보를 JSON 형태로 관리하고 <Card>에 주입합니다.
각 카드는 해양 생명체 일러스트와 함께 나타나도록 디자인합니다.
반응형 디자인
Tailwind CSS의 반응형 유틸리티를 활용해 모바일·태블릿·데스크톱에서 레이아웃이 자연스럽게 변하도록 합니다.
4.3 애니메이션과 시각 효과
로딩 애니메이션 – 진행 퍼센트가 0→100 %로 증가하면서 심해로 내려가는 듯한 효과를 구현합니다. 끝나면 슬로 모션으로 화면이 전환됩니다.
심해 스크롤 – 스크롤 위치에 따라 배경의 색과 이미지가 변하고, 해양 생명체 또는 장식이 느리게 움직이는 Parallax 효과를 적용합니다. Designmodo 기사에서 언급된 것처럼 “사용자를 스크롤로 유도”하는 역할을 합니다.
고래 인터랙션 – CodeTap 예제를 참조해 고래가 마우스를 따라 움직이거나 클릭 시 점프하는 애니메이션을 구현합니다. 소리 효과도 넣을 수 있도록 옵션을 둡니다.
카드 등장 애니메이션 – ScrollTrigger로 카드를 트리거하며 투명도와 위치를 변화시켜 부드럽게 등장시키고, 늦게 나타나는 요소들을 지연시켜 시선 흐름을 유도합니다.
4.4 테스트 및 개선
디바이스 테스트 – 다양한 브라우저(Chrome, Safari, Firefox)와 화면 크기에서 동작을 확인합니다.
퍼포먼스 최적화 – 이미지 용량을 줄이고, 애니메이션을 requestAnimationFrame을 이용해 최적화합니다. 필요 시 lazy loading을 도입합니다.
접근성 – 높은 대비 비율, 키보드 네비게이션, Reduced Motion 설정을 지원합니다.
사용자 피드백 – 초기 사용자 테스트를 통해 UX를 개선하고 애니메이션 강도나 길이를 조정합니다.
4.5 배포 및 운영
CI/CD 설정 – GitHub Actions를 이용해 커밋 시 자동 빌드 및 테스트를 수행하고, 성공 시 Vercel/Netlify로 배포합니다.
도메인 설정 – 자신만의 도메인을 연결해 브랜드를 강화합니다.
콘텐츠 업데이트 플랜 – 경력이나 프로젝트가 추가되면 JSON 데이터를 업데이트하고, 필요한 이미지를 추가합니다.

1. 로딩 애니메이션: 잠수 준비와 진행 퍼센트
컨셉과 시각적 특징
초기화 화면: Killian Herzer의 포트폴리오 사이트는 화면 진입 시 “INITIALIZING…”이라는 텍스트와 함께 여러 상태 메시지를 보여 주며, 신호를 찾는 레이더나 해커 콘솔을 연상시키는 분위기를 연출합니다. 이때 화면 곳곳에 무작위 숫자와 “SIGNAL TRACING” 같은 로그가 표시되어 긴장감을 높입니다.
진행 퍼센트: 화면 가운데에는 00%에서 시작해 100%까지 증가하는 퍼센트 표시가 나타납니다. 퍼센트 값은 흰색이나 네온 색상으로 강조되어 사용자의 시선을 끕니다.
슬로 모션 전환: 퍼센트가 100%에 도달하면 화면이 부드럽게 흐려지거나 확대되며 메인 콘텐츠로 넘어갑니다. 이 전환 과정에서 움직임을 슬로 모션처럼 느리게 처리하면 ‘잠수가 시작되는’ 느낌을 전달할 수 있습니다.
구현 방법
프로그레스 상태 관리
로딩할 자원(배경 이미지, 글꼴, Lottie 애니메이션 등)의 수를 미리 파악한 뒤 JavaScript에서 Promise.all()로 모두 읽어들입니다.
각 자원이 로드될 때마다 현재까지 완료된 수를 전체 수로 나눈 값을 상태에 저장하고, 퍼센트 텍스트를 업데이트합니다.
시각적 표현
배경색과 opacity 애니메이션: 퍼센트가 올라감에 따라 배경을 점점 어둡게 만들거나 물속으로 잠기는 듯한 블루 톤으로 서서히 변환합니다. 이를 위해 CSS 변수에 색 값을 저장하고, JavaScript에서 style.setProperty('--bg-color', 새로운값)으로 갱신할 수 있습니다.
숫자 애니메이션: 텍스트 요소의 innerText를 매 프레임마다 변경해 퍼센트 숫자가 카운트업 되도록 합니다. requestAnimationFrame을 사용하면 부드러운 업데이트가 가능합니다.
전환 효과
퍼센트가 100%가 되는 순간 className을 바꿔 전체 컨테이너에 transition: transform 1s ease-out; transform: scale(1.1); 같은 CSS를 적용하거나, opacity를 서서히 0으로 낮추어 메인 페이지가 나타나게 합니다.
이 시점에 배경음(바다에 잠수하는 소리나 고래 소리)을 페이드인 시켜 분위기를 조성할 수 있습니다.
2. 심해 스크롤: 깊이를 표현하는 패럴랙스와 색 변화
콘셉트와 참고 사례
사용자를 스크롤로 유도: Designmodo의 해양 테마 기사에서는 “바닷속 풍경이 사용자를 스크롤로 유도하며 탐험하도록 이끈다”고 설명합니다. 상단에는 밝은 해수면을, 아래로 내려갈수록 어두운 심해를 배경으로 배치해 자연스러운 여행을 제공합니다.
세로 스크롤로 깊어지는 연출: 예시 중 Luxaqua Design 같은 사이트는 “세로 스크롤을 기반으로 멋진 바다 깊숙이로 우리를 깊게 끌어들인다”고 평가됩니다. 스크롤을 내릴수록 배경 이미지와 색이 변하며 해양 생명체를 차례로 보여 줍니다.
구현 방법
섹션과 깊이 설정
웹 페이지를 여러 섹션으로 나누고, 각 섹션에 data-depth 속성을 부여합니다(예: 0m, 200m, 1000m 등). 깊이에 따라 색상과 콘텐츠가 달라집니다.
색상 변화: CSS custom property --bg-color를 사용해 각 섹션의 배경색을 설정하고, 스크롤에 따라 이 값을 interpolate하면서 점진적으로 짙은 파란색으로 변하도록 합니다.
패럴랙스 레이어 구성
배경 레이어, 중간 레이어(해초, 물고기 등), 전경 레이어(텍스트와 카드)를 각각 절대 위치로 배치합니다.
JavaScript 또는 GSAP의 ScrollTrigger를 사용해 스크롤 위치에 따라 각 레이어의 translateY 값을 다르게 변경합니다. SliderRevolution의 설명처럼 패럴랙스는 “배경 콘텐츠가 전경보다 느리게 이동하여 깊이감을 만든다”.
예를 들어 배경은 스크롤 값의 0.5배, 중간 레이어는 0.8배, 전경은 1배로 이동시켜 입체감을 줍니다.
해양 생명체 등장
각 섹션이 viewport에 들어올 때 해양 생물 SVG나 이미지를 살짝 움직이며 등장시키고, 천천히 떠다니도록 @keyframes float 애니메이션을 적용합니다.
Depth가 깊어질수록 다른 생물이 등장하도록 IntersectionObserver 또는 ScrollTrigger로 감지합니다.
3. 고래 인터랙션: 사용자와 상호작용하는 애니메이션
예시 분석
CodeTap 프로젝트: “고래 애니메이션 프로젝트는 HTML, CSS, JavaScript만으로 만들어지며, 클릭이나 호버 시 반응한다”고 설명합니다. CSS @keyframes와 transform으로 자연스러운 움직임을 만들고, JavaScript로 방향과 속도를 제어할 수 있다고 합니다.
커스터마이징: 프로젝트에서는 “버블이나 파도 효과를 추가하거나 고래의 색과 크기를 변경할 수 있고, 상호작용 시 사운드를 재생하는 등의 확장”을 제안합니다.
구현 방법
기본 구조
SVG 또는 <div> 요소로 고래 모양을 정의합니다. Tail과 Body를 별도 요소로 나누어 두 부분이 서로 약간 다른 속도로 움직이게 하면 자연스럽습니다.
CSS @keyframes swim에서 transform: translateX()와 rotate() 값을 조정해 고래가 앞뒤로 움직이고 꼬리를 흔드는 듯한 애니메이션을 만듭니다.
마우스 추적
React 컴포넌트 내부에서 onMouseMove 이벤트를 사용해 커서 좌표를 추적합니다.
requestAnimationFrame을 통해 고래의 translateX 및 translateY를 조금씩 커서 쪽으로 이동시킵니다. 이동 거리를 1/10씩 보간(interpolate)하면 추격하는 듯한 부드러운 움직임을 얻을 수 있습니다.
클릭/호버 반응
onClick 이벤트에서 점프 애니메이션을 트리거합니다. 예를 들어 @keyframes jump를 정의하고 animation: jump 1s ease-out;를 적용합니다.
소리 효과를 위해 <audio> 태그를 숨겨두고, 클릭 시 audio.play()를 호출하도록 합니다. 사용자가 원하지 않을 경우를 대비해 음소거 옵션을 제공합니다.
반응형 디자인
화면 크기가 작을 때에는 고래의 크기를 줄이고 마우스 대신 터치 이벤트에 반응하도록 합니다. pointermove 이벤트를 사용해 터치 환경에서도 동일하게 동작하도록 구현합니다.
4. 카드 등장 애니메이션: 정보의 단계적 노출
개념과 기술
ScrollTrigger를 이용한 등장: GSAP의 ScrollTrigger는 지정한 요소가 뷰포트에 들어올 때 애니메이션을 실행하도록 도와줍니다. 카드처럼 여러 요소가 있는 경우 등장 시간을 약간씩 지연시키면 부드러운 흐름을 만들 수 있습니다.
CSS 스크롤 애니메이션 대안: BigDevSoon 사이트는 CSS만으로 스크롤 진입 애니메이션을 구현하는 예제를 소개합니다. .card에 animation: reveal linear both; animation-timeline: view(); animation-range: entry 0% entry 100%;를 사용해 화면에 들어올 때 opacity와 Y위치를 애니메이션합니다. 이러한 방식은 JavaScript가 필요 없어 성능과 접근성에서 장점이 있습니다.
구현 방법
GSAP ScrollTrigger 사용 시

각 카드에 고유한 ref를 부여하고, 컴포넌트 마운트 시 gsap.from() 또는 gsap.to()를 사용해 초기 상태와 최종 상태를 정의합니다. 예를 들어:

cards.forEach((card, i) => {
  gsap.from(card, {
    scrollTrigger: {
      trigger: card,
      start: 'top 80%', // 뷰포트 80% 지점에서 시작
    },
    opacity: 0,
    y: 40,
    duration: 0.6,
    delay: i * 0.15 // 카드 순서에 따른 지연
  });
});
이렇게 하면 첫 번째 카드는 즉시 나타나고 다음 카드는 0.15초씩 지연돼 연속적인 등장 효과가 납니다.
CSS view() 애니메이션 사용 시
최신 브라우저(Chrome 115+)에서는 animation-timeline: view()를 사용할 수 있습니다. 앞선 예제처럼 .card 클래스에 opacity와 transform: translateY()를 애니메이션하는 @keyframes reveal을 정의하고 animation-range를 이용해 요소가 완전히 나타날 때까지 애니메이션을 진행합니다.
JavaScript 의존성이 없으므로 퍼포먼스가 뛰어나며, 다만 Firefox 등 일부 브라우저에서는 플래그를 켜야 동작합니다.
시선 흐름 유도
카드 내부 콘텐츠도 시간차를 두어 나타나게 하면 사용자가 하나씩 내용을 읽도록 유도할 수 있습니다. 예를 들어 제목, 설명, 버튼을 각각 다른 시점에 나타내도록 설정합니다.
카드 배경에 subtle한 그림자와 서서히 밝아지는 효과를 추가해 깊이감을 더합니다.
5. 종합 팁 및 최적화
과용을 피하기: Designmodo 기사에서는 해양 테마가 강력한 감정 효과를 불러일으킨다고 언급하지만, 과도한 애니메이션은 혼란을 야기할 수 있습니다. 패럴랙스나 인터랙션 효과는 핵심 요소에만 적용하여 사용자 경험을 흐리지 않도록 합니다.
접근성 고려: 시각적으로 화려한 효과를 제공하더라도, 설정에서 “Reduced Motion”을 선택한 사용자를 위해 애니메이션을 최소화할 수 있도록 옵션을 제공합니다. 또한 충분한 색 대비와 명확한 텍스트 크기를 유지합니다.
성능 최적화: 이미지와 SVG는 WebP나 압축된 형태로 제공하고, will-change 속성을 사용해 CSS 애니메이션을 GPU 레이어에서 처리합니다. requestAnimationFrame을 통해 애니메이션을 최적화하고, 스크롤 이벤트는 쓰로틀링(throttling)으로 과도한 호출을 방지합니다.
